❯ 项目中的Session ID是以什么作为唯一值的                                        

● 根据项目文档，Session ID 的唯一值来源如下：

  ┌───────────┬─────────────────┬────────────────────────────────────────────┐
  │ 消息类型  │ Session ID 格式 │                    说明                    │
  ├───────────┼─────────────────┼────────────────────────────────────────────┤
  │ 私聊消息  │ c2c_xxx         │ 使用用户的 openid 作为唯一标识             │
  ├───────────┼─────────────────┼────────────────────────────────────────────┤
  │ 群聊@消息 │ group_xxx       │ 使用群的 openid 作为唯一标识，群内共享历史 │
  └───────────┴─────────────────┴────────────────────────────────────────────┘

  具体实现在 src/bot_client.py 中：
  - on_c2c_message_create() - 私聊消息，使用 c2c_ + 用户openid
  - on_group_at_message_create() - 群聊@消息，使用 group_ + 群openid

  这样设计的目的是让同一个用户在私聊时有独立的会话上下文，而群聊中所有成员共享同
  一个会话历史。
