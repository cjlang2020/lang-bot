# 工具模块重构说明

## 变更日期
2026-03-11

## 变更概述

将原有的单文件 `src/windows_tools.py`（700+行）重构为模块化的工具系统，按功能分类到 `src/tools/` 目录下。

## 变更详情

### 1. 代码结构变更

**重构前：**
```
src/
└── windows_tools.py  (700+ 行，包含所有工具)
```

**重构后：**
```
src/
├── windows_tools.py  (向后兼容，仅导入)
└── tools/
    ├── __init__.py
    ├── file_system.py    (~180 行)
    ├── search.py         (~160 行)
    ├── system.py         (~150 行)
    ├── network.py        (~70 行)
    ├── time.py           (~40 行)
    └── tool_registry.py  (~70 行)
```

### 2. 功能分类

| 模块 | 功能 | 工具数量 |
|------|------|----------|
| file_system.py | 文件系统操作 | 4 |
| search.py | 文件搜索 | 2 |
| system.py | 系统操作 | 3 |
| network.py | 网络操作 | 2 |
| time.py | 时间操作 | 1 |

**总计**：12个工具

### 3. 向后兼容性

保留了 `src/windows_tools.py` 作为向后兼容的导入文件，内容简化为：

```python
from src.tools import (
    TOOLS,
    TOOL_FUNCTIONS,
    process_tool_calls
)
```

这意味着：
- ✅ 现有代码无需修改
- ✅ ai_client.py 等模块可以继续使用 `from src.windows_tools import ...`
- ✅ 新代码建议使用 `from src.tools import ...`

### 4. 新增功能

**搜索工具升级：**
- 从 Everything 框架迁移到 Ripgrep
- 新增 `search_content` 工具，支持按内容搜索
- 改进编码处理，解决中文文件名乱码问题
- 添加完整的错误处理和回退机制

**模块化优势：**
- 每个工具模块独立，易于理解和维护
- 添加新工具只需在对应模块中添加
- 更好的测试支持
- 更清晰的代码组织

## 使用方法

### 对于现有代码

无需修改，继续使用：
```python
from src.windows_tools import TOOLS, TOOL_FUNCTIONS, process_tool_calls
```

### 对于新代码

建议使用：
```python
from src.tools import TOOLS, TOOL_FUNCTIONS, process_tool_calls
```

### 添加新工具

参考 `src/tools/README.md` 中的"扩展工具"章节。

## 测试建议

1. **启动测试**：验证程序能否正常启动
```bash
python start_listener.py
```

2. **搜索功能测试**：
```bash
python test_ripgrep.py
```

3. **工具调用测试**：
   - 发送 `/会话` 查看可用工具
   - 测试 `list_directory`、`read_file` 等基本工具
   - 测试 `search_files`、`search_content` 搜索功能
   - 测试 `execute_command` 等系统工具

## 注意事项

1. **Ripgrep依赖**：使用搜索功能需要安装 Ripgrep
2. **编码问题**：已修复中文文件名乱码问题
3. **向后兼容**：windows_tools.py 保留为导入文件，建议新代码使用 src.tools
4. **工具名称**：所有工具名称必须唯一，已在各模块中确保

## 优势总结

1. **可维护性提升**：代码行数从700+减少到每个模块100-200行
2. **可扩展性提升**：添加新工具更加简单
3. **可读性提升**：代码结构清晰，功能分类明确
4. **可测试性提升**：可以单独测试每个工具模块
5. **向后兼容**：现有代码无需修改
