## 🔧 技术细节

### 1. 统一接口

所有工具通过 `tool_registry.py` 统一管理：

```python
from src.tools import TOOLS, TOOL_FUNCTIONS, process_tool_calls

# TOOLS: 工具定义列表（用于AI Function Calling）
# TOOL_FUNCTIONS: 工具名称到函数的映射
# process_tool_calls: 工具调用处理函数
```

### 2. 模块导入

`tool_registry.py` 自动聚合所有工具：

```python
from .file_system import TOOL_DEFINITIONS as FILE_SYSTEM_TOOLS
from .search import TOOL_DEFINITIONS as SEARCH_TOOLS
# ... 其他模块

TOOLS = FILE_SYSTEM_TOOLS + SEARCH_TOOLS + ...
TOOL_FUNCTIONS = {**FILE_SYSTEM_FUNCTIONS, **SEARCH_FUNCTIONS, ...}
```

### 3. 异步设计

所有工具函数都是异步的：
```python
async def list_directory_tool(path: str, ...) -> str:
    ...
```

1. **Ripgrep安装**：使用搜索功能需要安装 `rg` 命令行工具
   ```bash
   scoop install ripgrep  # 或 choco install ripgrep
   ```

2. **编码问题**：已修复，但需要注意：
   - 所有 subprocess 调用都使用 UTF-8 编码
   - 错误处理使用 `errors='replace'` 避免崩溃

3. **向后兼容**：
   - 现有代码使用 `src.windows_tools` 不受影响
   - 新代码建议使用 `src.tools`

4. **工具名称唯一性**：所有工具的 `name` 字段必须唯一
